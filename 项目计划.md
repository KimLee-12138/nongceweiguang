# P0 基线冻结计划

## 时间与目标

- 时间窗：2026-04-11 至 2026-04-12
- 联调基线：MySQL 为唯一权威联调库，SQLite 仅保留本地开发回退用途
- 目标：完成范围冻结、接口冻结、数据表冻结、12 功能任务清单、联调环境补齐、演示数据补齐、外部能力验证入口

## 当前结论

- 已确认 `backend/.env` 中的 MySQL 可连通，API 启动时可读取现有 MySQL 表。
- 前后端主干页面与接口已具备联调骨架，但之前存在模型编译、审核 AI、OCR、真实湖北政策抓取、部分洞察/风向标统计为占位实现的问题。
- 本轮已将依赖检查、真实抓取通路、OCR 通路、模型 provider、自动爬虫 run 记录、演示数据写入命令纳入实现范围。
- 若 `DEEPSEEK_API_KEY` 或 `BAIDU_OCR_*` 未配置，则 P0 验收应判定为“能力未验真”，不能视为完成。

## 冻结口径

### 环境

- API：`http://localhost:8000/api/v1`
- 前端：`http://localhost:5173`
- 调度器：独立执行 `python -m app.scheduler`
- 依赖检查：
  - `GET /api/v1/health`
  - `GET /api/v1/health/dependencies?live=true`
  - `python -m app.cli verify-baseline --live`

### 冻结接口

- 鉴权：`/auth/*`、`/user-auth/*`
- 画像：`/profiles`、`/profiles/{id}`、`/profiles/{id}/suggested-policies`
- 匹配：`/policies/evaluate`、`/policies/{policy_id}/match-for-profile/{profile_id}`
- 聊天：`/chat/conversations*`、`/chat/stream`
- 政策管理：`/policies*`、`/policies/review/*`
- 作业与运维：`/admin-ops/*`
- 洞察：`/insights/*`
- 风向标：`/compass/*`

### 冻结数据表

- 鉴权与限流：`admin_users`、`end_users`、`auth_sessions`、`auth_refresh_tokens`、`request_rate_limits`
- 业务主表：`user_profiles`、`policies`、`match_records`、`chat_conversations`、`chat_messages`
- 审核与原始数据：`policy_review_tasks`、`policy_review_events`、`hubei_policies_raw`
- 洞察与风向标：`compass_reports`、`compass_glossary`
- 配置与作业：`system_config`、`admin_operation_runs`、`admin_operation_run_items`、`auto_crawler_runs`、`telemetry_events`
